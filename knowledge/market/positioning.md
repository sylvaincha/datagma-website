# Datagma — Positionnement marché

## Positionnement actuel (site de 5 ans)
"B2B Data Enrichment & Phone Finder" — généraliste

## Nouveau positionnement recommandé

### Ligne directrice
**"The Phone Finder that actually finds phones"**
Ou : **"Mobile-first B2B enrichment — find numbers nobody else has"**

Le marché du data enrichment est encombré (Apollo, Hunter, Lusha, Kaspr...).
Datagma doit se concentrer sur ce qui le rend unique :
1. **Plus de numéros mobiles que n'importe qui d'autre** (benchmark prouvé)
2. **Uniquement des mobiles** (pas de faux positifs/landlines)
3. **Recherche sans URL LinkedIn** (nom + pays = c'est tout)

## Évolution du marché (2024-2026)

### L'émergence du Waterfall Enrichment
Le waterfall enrichment est devenu le standard en RevOps :
- On interroge plusieurs providers en séquence
- On garde le premier résultat valide
- On optimise le coût par donnée enrichie

**Datagma est idéalement positionné dans un waterfall** car :
- API simple (GET calls uniquement)
- Prix compétitif
- Spécialisé téléphones (là où les autres échouent)
- Compatible Clay, Fullenrich, BetterContact, GetCargo

### Clay et l'essor des "clay tables"
Clay est devenu le hub central de l'outbound moderne.
- Des milliers d'équipes Sales utilisent Clay comme orchestrateur
- Datagma est un provider natif de Clay
- Opportunité : créer du contenu "How to use Datagma in Clay"

### L'impact de l'IA sur la prospection
- Les outils IA (ChatGPT, Claude) sont utilisés pour créer des messages personnalisés
- La donnée de contact reste le bottleneck → opportunité pour Datagma
- Les LLMs indexent les pages de documentation → llms.txt important

## Segments de marché

### 1. Outbound Sales (SDR/BDR/AE)
- Besoin : trouver les bons numéros pour appeler
- Pain : 40-60% des numéros sont des fixes inutiles
- Solution Datagma : uniquement des mobiles, taux de match 70-77%

### 2. Growth / RevOps avec Clay
- Besoin : waterfall enrichment automatisé
- Pain : les providers sont chers et peu fiables sur les téléphones
- Solution Datagma : API native Clay, prix optimal, spécialiste téléphone

### 3. Développeurs / SaaS
- Besoin : enrichir les signups avec données entreprise
- Pain : API complexes, prix élevés, GDPR
- Solution Datagma : GET calls simple, GDPR by design, 75+ data points

### 4. Recruteurs
- Besoin : trouver les coordonnées des candidats LinkedIn
- Pain : LinkedIn ne donne pas les coordonnées
- Solution Datagma : Chrome Extension + API + File Upload

## Tone of voice recommandé

### Principe
- **Honnête et direct** : on a des benchmarks, on les montre
- **Technique mais accessible** : les acheteurs sont des RevOps/Growth, pas uniquement des managers
- **Data-driven** : chiffres précis, pas de sur-promesse (77%, pas "le meilleur")
- **Premium assumé** : le prix est plus élevé car la qualité est supérieure

### Ce qu'on NE dit PAS
- "Le seul outil qui..." (sur-promesse)
- "Révolutionnaire" ou "game-changing"
- "Simple et facile" sans prouver

### Ce qu'on DIT
- "77% de taux de match en Inde, vs 44% pour Lusha"
- "Uniquement des mobiles — pas de landlines"
- "Trouvez un contact avec son prénom, nom et pays — sans LinkedIn URL"
- "GDPR by design : on ne stocke pas de données personnelles"
